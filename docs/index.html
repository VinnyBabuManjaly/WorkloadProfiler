Executive Summary

Modern compute clusters run hundreds of thousands of workloads — but most infrastructure teams treat them as homogeneous. This project builds a scalable, data-driven framework to automatically discover distinct workload types using real production trace data from Google’s 2019 cluster sample.

By analyzing CPU usage, memory behavior, runtime characteristics, and resource efficiency, the system identifies behavioral workload segments that support:

  • Smarter autoscaling strategies
  • Reduced overprovisioning
  • Better SLA risk visibility
  • Behavior-aware scheduling policies

Business Problem

Infrastructure teams face:

  • Overprovisioned workloads wasting memory
  • Underutilized CPU allocations
  • High failure rates without behavioral segmentation
  • Runtime-based scheduling that ignores resource patterns

The central question: Can we automatically discover natural workload categories that improve resource efficiency and SLA management?

Technical Approach

1. Feature Engineering

  • Runtime calculation
  • CPU & memory utilization ratios
  • Overprovisioning metrics
  • Peak-to-average burst ratios
  • Page cache dependency

2. Dimensionality Reduction

  • PCA applied to reduce noise
  • PC1 + PC2 explain ~54% variance

3. Clustering

  • Baselines: Single Cluster & Runtime Quantile
  • Primary Model: HDBSCAN (density-based)
  • Validation: DBSCAN
